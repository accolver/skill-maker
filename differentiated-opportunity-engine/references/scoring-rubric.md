# Scoring Rubric

Score each opportunity from 1-10 on the dimensions below. Use the scores to
compare candidates, not as false precision.

Before scoring, run a hard screen for:

- Illegal or non-compliant tactics
- Employer confidentiality or IP conflicts
- Constraint bundles that make the path unrealistic right now

If a candidate fails the hard screen, reject it or mark the overall answer as
`conditional` or `no_go`.

## Primary dimensions

### 1. Edge fit

- `9-10`: Strong domain advantage, trusted access, or proprietary workflow
  knowledge
- `6-8`: Clear but not exclusive advantage
- `3-5`: Some relevance but weak differentiation
- `1-2`: Generic opportunity with no meaningful edge

### 2. Speed to revenue

- `9-10`: Can plausibly test paid demand within 7-14 days
- `6-8`: Likely needs light setup or outreach before revenue
- `3-5`: Requires meaningful build or audience work first
- `1-2`: Long path before any proof of demand

### 3. Durability

- `9-10`: Compounds into software, data, repeatable process, or brand asset
- `6-8`: Moderate repeatability or retention
- `3-5`: Mostly transactional
- `1-2`: Fragile or trend-dependent

### 4. Distribution fit

- `9-10`: User already has direct access to buyers or a strong trusted channel
- `6-8`: Reachable through warm intros or obvious niche channels
- `3-5`: Requires cold start distribution effort
- `1-2`: No credible route to initial customers

### 5. AI leverage

- `9-10`: AI materially improves cost, speed, or quality of delivery
- `6-8`: Useful advantage but not the full moat
- `3-5`: AI is mostly cosmetic
- `1-2`: AI does not change the economics much

### 6. Defensibility

- `9-10`: Protected by access, data, workflow ownership, relationships, or
  reputation
- `6-8`: Some moat from specialization or distribution
- `3-5`: Easy to copy
- `1-2`: Commodity wrapper or obvious clone

## Penalty dimensions

### 7. Execution difficulty

- `1-2`: Simple to start manually
- `3-5`: Moderate setup or coordination burden
- `6-8`: Significant build or delivery complexity
- `9-10`: Hard to launch given the user's current resources

### 8. Compliance and platform risk

- `1-2`: Low legal and platform risk
- `3-5`: Some policy or operational sensitivity
- `6-8`: High risk of restrictions, policy changes, or compliance burden
- `9-10`: Not suitable for recommendation without specialist review

## Ranking guidance

- A weak `edge fit` should usually eliminate or severely downgrade the idea.
- A very strong `speed to revenue` score should not rescue an idea with no
  defensibility.
- Prefer `hybrid` opportunities when scores are otherwise close, not when the
  user's constraints clearly rule them out.
- Use `content-first` opportunities only when `distribution fit` is unusually
  strong.
- If the user's preferred path loses on edge, distribution, or legality, say so
  explicitly.
- If the user wants micro-SaaS but lacks proof of demand, check whether the same
  problem can start as a paid service or concierge workflow.
- If no candidate is credible under the current constraints, return
  `conditional` or `no_go` rather than forcing a weak recommendation.
