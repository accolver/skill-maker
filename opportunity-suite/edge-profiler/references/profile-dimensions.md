# Profile Dimensions

Use these dimensions when building the user profile.

## Core fields

- `skills`: practical capabilities the user can actually deploy
- `domain_experience`: niche or industry familiarity
- `assets`: audience, datasets, tooling, brands, communities, relationships
- `distribution`: channels the user already controls or can credibly access
- `network_access`: warm intros, buyer access, partner access
- `constraints`: time, capital, legal, preference, geography, risk tolerance
- `credibility_signals`: title, years of experience, track record, trust markers
- `unfair_advantages`: meaningful advantages over a generic competent builder

## Edge classification

### Usable edge

Edge the user can legally and practically exploit now.

Examples:

- Warm buyer network
- Trusted niche audience
- Real workflow familiarity
- Publicly usable expertise

### Restricted or unusable edge

Edge that looks attractive but should not be treated as usable without more
proof.

Examples:

- Employer IP or templates
- Customer access the user cannot solicit
- PHI or sensitive regulated data
- Licensed authority the user does not hold

## Confidence guidance

- `high`: user gave direct evidence
- `medium`: strong implication but still some uncertainty
- `low`: assumption used because context was thin
