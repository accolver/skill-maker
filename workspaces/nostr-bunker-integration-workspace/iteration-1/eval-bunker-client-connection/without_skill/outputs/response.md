# Nostr Bunker (NIP-46) Remote Signer Integration with @nostr/tools

This guide covers the complete flow for connecting to a NIP-46 remote signer via
a `bunker://` URI using `@nostr/tools`, including session persistence across
page reloads.

## Overview

NIP-46 defines a protocol for remote signing. Instead of the client holding the
user's private key, a separate "remote signer" (bunker) holds it. The client
communicates with the bunker over Nostr relays using encrypted kind:24133
events. The user provides a `bunker://` URI that contains the remote signer's
pubkey and relay information.

**Key concepts:**

- **client-keypair**: Ephemeral keys generated by your app for encrypted
  communication with the bunker
- **remote-signer-pubkey**: The bunker's pubkey (from the `bunker://` URI) —
  this may differ from the user's pubkey
- **user-pubkey**: The actual user's pubkey, retrieved via `get_public_key`
  after connecting

## Complete Implementation

### 1. Bunker Connection Service (`bunker-service.ts`)

```typescript
import { BunkerSigner, parseBunkerInput } from "@nostr/tools/nip46";
import { SimplePool } from "@nostr/tools/pool";
import { generateSecretKey } from "@nostr/tools/pure";
import { bytesToHex, hexToBytes } from "@noble/hashes/utils";

/**
 * Session data persisted to localStorage so users stay logged in
 * across page reloads.
 */
interface BunkerSession {
  /** Hex-encoded local secret key used for NIP-44 encrypted communication */
  localSecretKeyHex: string;
  /** The original bunker:// URI the user pasted */
  bunkerUri: string;
  /** The user's actual pubkey (retrieved via get_public_key) */
  userPubkey: string;
}

const SESSION_STORAGE_KEY = "nostr_bunker_session";

/**
 * BunkerConnectionManager handles the full lifecycle of a NIP-46
 * remote signer connection:
 * - Parsing bunker:// URIs
 * - Establishing connections
 * - Retrieving the user's pubkey
 * - Signing events
 * - Persisting and restoring sessions
 */
export class BunkerConnectionManager {
  private pool: SimplePool;
  private signer: BunkerSigner | null = null;
  private userPubkey: string | null = null;

  constructor() {
    this.pool = new SimplePool();
  }

  /**
   * Connect to a remote signer using a bunker:// URI pasted by the user.
   *
   * Flow:
   * 1. Parse the bunker:// URI to extract remote-signer-pubkey and relays
   * 2. Generate a local (client) secret key for encrypted communication
   * 3. Create a BunkerSigner and call connect()
   * 4. Retrieve the user's actual pubkey via get_public_key
   * 5. Persist session data to localStorage
   *
   * @param bunkerUri - The bunker:// URI from the remote signer
   * @returns The user's public key (hex)
   */
  async connectFromUri(bunkerUri: string): Promise<string> {
    // Step 1: Parse the bunker URI
    const bunkerPointer = await parseBunkerInput(bunkerUri);
    if (!bunkerPointer) {
      throw new Error(
        "Invalid bunker URI. Expected format: bunker://<pubkey>?relay=wss://...",
      );
    }

    // Step 2: Generate a fresh local secret key for this session
    const localSecretKey = generateSecretKey();

    // Step 3: Create the BunkerSigner and connect
    this.signer = BunkerSigner.fromBunker(localSecretKey, bunkerPointer, {
      pool: this.pool,
    });

    try {
      await this.signer.connect();
    } catch (err) {
      this.signer = null;
      throw new Error(
        `Failed to connect to bunker: ${
          err instanceof Error ? err.message : String(err)
        }`,
      );
    }

    // Step 4: Get the user's actual public key
    // IMPORTANT: The remote-signer-pubkey from the URI may differ from the
    // user's pubkey. Always call getPublicKey() after connecting.
    this.userPubkey = await this.signer.getPublicKey();

    // Step 5: Persist session for page reload survival
    const session: BunkerSession = {
      localSecretKeyHex: bytesToHex(localSecretKey),
      bunkerUri,
      userPubkey: this.userPubkey,
    };
    localStorage.setItem(SESSION_STORAGE_KEY, JSON.stringify(session));

    return this.userPubkey;
  }

  /**
   * Restore a previously established session from localStorage.
   * Call this on app startup to keep users logged in across reloads.
   *
   * @returns The user's pubkey if session was restored, null otherwise
   */
  async restoreSession(): Promise<string | null> {
    const stored = localStorage.getItem(SESSION_STORAGE_KEY);
    if (!stored) {
      return null;
    }

    let session: BunkerSession;
    try {
      session = JSON.parse(stored);
    } catch {
      // Corrupted session data — clear it
      localStorage.removeItem(SESSION_STORAGE_KEY);
      return null;
    }

    // Reconstruct the connection using the persisted local secret key
    // and the original bunker URI
    const bunkerPointer = await parseBunkerInput(session.bunkerUri);
    if (!bunkerPointer) {
      localStorage.removeItem(SESSION_STORAGE_KEY);
      return null;
    }

    const localSecretKey = hexToBytes(session.localSecretKeyHex);

    this.signer = BunkerSigner.fromBunker(localSecretKey, bunkerPointer, {
      pool: this.pool,
    });

    try {
      await this.signer.connect();
    } catch (err) {
      // Connection failed — session may be expired or bunker offline
      console.warn("Failed to restore bunker session:", err);
      localStorage.removeItem(SESSION_STORAGE_KEY);
      this.signer = null;
      return null;
    }

    // Re-verify the user's pubkey
    this.userPubkey = await this.signer.getPublicKey();

    return this.userPubkey;
  }

  /**
   * Sign a kind:1 text note using the remote signer.
   *
   * The BunkerSigner sends a sign_event request (kind:24133) to the
   * remote signer, which signs it with the user's actual private key
   * and returns the fully signed event.
   *
   * @param content - The text content of the note
   * @param tags - Optional tags for the event
   * @returns The fully signed Nostr event
   */
  async signNote(
    content: string,
    tags: string[][] = [],
  ): Promise<{
    id: string;
    pubkey: string;
    created_at: number;
    kind: number;
    tags: string[][];
    content: string;
    sig: string;
  }> {
    if (!this.signer) {
      throw new Error("Not connected to bunker. Call connectFromUri() first.");
    }

    const signedEvent = await this.signer.signEvent({
      kind: 1,
      created_at: Math.floor(Date.now() / 1000),
      tags,
      content,
    });

    return signedEvent;
  }

  /**
   * Publish a signed note to relays.
   *
   * @param relays - Array of relay URLs to publish to
   * @param content - The text content of the note
   * @param tags - Optional tags
   */
  async publishNote(
    relays: string[],
    content: string,
    tags: string[][] = [],
  ): Promise<void> {
    const signedEvent = await this.signNote(content, tags);
    await Promise.any(this.pool.publish(relays, signedEvent));
  }

  /**
   * Get the connected user's public key.
   */
  getConnectedPubkey(): string | null {
    return this.userPubkey;
  }

  /**
   * Check if there's an active bunker connection.
   */
  isConnected(): boolean {
    return this.signer !== null && this.userPubkey !== null;
  }

  /**
   * Log out: close the bunker connection and clear the persisted session.
   */
  async logout(): Promise<void> {
    if (this.signer) {
      await this.signer.close();
      this.signer = null;
    }
    this.userPubkey = null;
    localStorage.removeItem(SESSION_STORAGE_KEY);
    this.pool.close([]);
    this.pool = new SimplePool();
  }

  /**
   * Clean up resources. Call when the app is unmounting.
   */
  async destroy(): Promise<void> {
    if (this.signer) {
      await this.signer.close();
    }
    this.pool.close([]);
  }
}
```

### 2. Handling Auth Challenges

NIP-46 defines an "auth challenge" mechanism. When the remote signer needs the
user to authenticate through an external method (e.g., a web page), it responds
with `result: "auth_url"` and an `error` field containing a URL. The client
should open this URL for the user.

`@nostr/tools`' `BunkerSigner` handles auth challenges internally via a
callback. You can provide an `onauth` handler when creating the signer:

```typescript
import { BunkerSigner, parseBunkerInput } from "@nostr/tools/nip46";
import { SimplePool } from "@nostr/tools/pool";
import { generateSecretKey } from "@nostr/tools/pure";

async function connectWithAuthHandling(
  bunkerUri: string,
): Promise<BunkerSigner> {
  const bunkerPointer = await parseBunkerInput(bunkerUri);
  if (!bunkerPointer) {
    throw new Error("Invalid bunker URI");
  }

  const pool = new SimplePool();
  const localSecretKey = generateSecretKey();

  const signer = BunkerSigner.fromBunker(localSecretKey, bunkerPointer, {
    pool,
    // Called when the remote signer requires user authentication
    // via an external URL (auth challenge per NIP-46)
    onauth: (authUrl: string) => {
      // Option A: Open in a popup window
      const popup = window.open(authUrl, "nostr-auth", "width=600,height=700");
      if (!popup) {
        // Popup blocked — fall back to redirect or show link
        console.warn("Popup blocked. Please visit:", authUrl);
        // You could show a modal with the URL for the user to click
      }

      // Option B: Open in a new tab
      // window.open(authUrl, '_blank')

      // Option C: Show a modal with the URL (for mobile-friendly UX)
      // showAuthModal(authUrl)
    },
  });

  await signer.connect();
  return signer;
}
```

### 3. Updated Connection Manager with Auth Challenge Support

Here's the enhanced version of the connection manager that integrates auth
challenge handling:

```typescript

  bunkerUri: string;
  userPubkey: string;
}

type AuthChallengeHandler = (url: string) => void;

export class BunkerConnectionManager {
  private pool: SimplePool;
  private signer: BunkerSigner | null = null;
  private userPubkey: string | null = null;
  private onAuthChallenge: AuthChallengeHandler;

  /**
   * @param onAuthChallenge - Callback invoked when the remote signer
   *   requires the user to authenticate via an external URL.
   *   The NIP-46 spec says the client should display this URL
   *   (popup or new tab) and then wait for the signer's response.
   */

    this.pool = new SimplePool();
    this.onAuthChallenge = onAuthChallenge ?? ((url: string) => {
      window.open(url, "nostr-auth", "width=600,height=700");

      pool: this.pool,
      onauth: this.onAuthChallenge,
    });

    try {
      await this.signer.connect();
      this.userPubkey = await this.signer.getPublicKey();
      return this.userPubkey;
    } catch (err) {
      console.warn("Failed to restore bunker session:", err);
      localStorage.removeItem(SESSION_STORAGE_KEY);

      return null;
    }

    }
    this.pool.close([]);
  }
```

### 4. Usage Example — Login Page Component

```typescript
// Example usage in a UI component (framework-agnostic logic)

const bunkerManager = new BunkerConnectionManager((authUrl) => {
  // Show auth challenge URL to user — this fires when the remote
  // signer needs the user to authenticate externally
  const popup = window.open(authUrl, "nostr-auth", "width=600,height=700");
  if (!popup) {
    alert(`Please open this URL to authenticate: ${authUrl}`);
  }
});

// ── On app startup: try to restore existing session ──
async function initApp() {
  const pubkey = await bunkerManager.restoreSession();
  if (pubkey) {
    console.log("Session restored! User pubkey:", pubkey);
    showLoggedInUI(pubkey);
  } else {
    showLoginForm();
  }
}

// ── When user pastes a bunker:// URI and clicks "Connect" ──
async function handleLogin(bunkerUri: string) {
  try {
    const pubkey = await bunkerManager.connectFromUri(bunkerUri);
    console.log("Connected! User pubkey:", pubkey);
    showLoggedInUI(pubkey);
  } catch (err) {
    console.error("Login failed:", err);
    showError(err instanceof Error ? err.message : "Connection failed");
  }
}

// ── When user clicks "Post" ──
async function handlePost(content: string) {
  try {
    const signedEvent = await bunkerManager.signNote(content);
    console.log("Signed event:", signedEvent);

    // Publish to relays
    const relays = ["wss://relay.damus.io", "wss://nos.lol"];
    await bunkerManager.publishNote(relays, content);
    console.log("Published!");
  } catch (err) {
    console.error("Failed to sign/publish:", err);
  }
}

async function handleLogout() {

}
```

## How Session Persistence Works

The session persistence strategy stores three pieces of data in `localStorage`:

1. **`localSecretKeyHex`** — The hex-encoded client secret key. This is the key
   used for NIP-44 encrypted communication with the bunker. By reusing the same
   key, the bunker recognizes the client on reconnection.

2. **`bunkerUri`** — The original `bunker://` URI. This contains the remote
   signer's pubkey and relay URLs needed to re-establish the connection.

3. **`userPubkey`** — The user's actual public key (cached for immediate UI
   display before reconnection completes).

On page reload, `restoreSession()` reconstructs the `BunkerSigner` with the same
local secret key and bunker pointer, then calls `connect()` again. The bunker
recognizes the returning client key and re-establishes the session without
requiring the user to re-paste the URI.

### Security Considerations

- The `localSecretKey` stored in localStorage is **not** the user's private key
  — it's an ephemeral client key used only for encrypted communication with the
  bunker. The user's actual private key never leaves the remote signer.
- For higher security, consider using `sessionStorage` instead (clears when the
  tab closes) or encrypting the stored session data.
- Always clear the session on logout via `localStorage.removeItem()`.

## Key Points

1. **Always call `getPublicKey()` after `connect()`** — The pubkey in the
   `bunker://` URI is the _remote signer's_ pubkey, which may differ from the
   _user's_ pubkey. The NIP-46 spec requires calling `get_public_key` to learn
   the actual user pubkey.

2. **Auth challenges are async** — When the bunker needs external
   authentication, it sends back an `auth_url`. The `onauth` callback fires, you
   show the URL to the user, and the library automatically waits for the
   bunker's follow-up response on the same request ID.

3. **Communication uses kind:24133 events** — All requests and responses between
   client and bunker are NIP-44 encrypted kind:24133 events sent over the relays
   specified in the bunker URI.

4. **The client key is not the user's key** — The `localSecretKey` is an
   ephemeral key for encrypted transport. The user's actual private key never
   leaves the remote signer. This is the entire security model of NIP-46.

5. **Session persistence requires the same client key** — To restore a session,
   you must reuse the same `localSecretKey` that was used during the initial
   connection. This is why we persist it (hex-encoded) in localStorage.
