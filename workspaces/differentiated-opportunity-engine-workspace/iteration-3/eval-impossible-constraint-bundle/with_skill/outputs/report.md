# Best Bet

Do not try to hit $3000 MRR from a new AI micro-SaaS in 30 days under the
current constraints. The correct move is to change exactly one major constraint
now and use the next 30 days to build edge, not chase MRR.

The single best change is: allow one low-friction distribution channel.

The best version of that is:

- Pick one narrow niche you already touch naturally, even if it is small. Good
  examples: job application tracking, freelance proposal cleanup, QA bug triage,
  recruiting follow-up, or e-commerce CSV cleanup.
- Allow async buyer contact in one place only. This can be forum replies,
  community DMs, marketplace listings, or email. It does not require calls.
- Sell or pre-sell one tiny outcome, not a broad AI app.

Why the obvious alternative loses:

- Pure micro-SaaS-first loses because there is no edge, no buyer access, and not
  enough time for iteration.
- Pure content loses because the user refuses content and has no audience.
- Pure service usually wins for speed, but the user refuses services.

What would change the recommendation:

- If the user allows 5 to 10 async customer conversations, a narrow
  service-to-tool ladder becomes viable.
- If the user allows 6 to 8 hours per week instead of 2, a marketplace
  micro-tool becomes more plausible.
- If the user already has access to a niche community, the answer shifts from
  `no_go` to `conditional`.

# 7-Day Constraint-Change Plan

The goal for the next 7 days is not revenue. It is to create the minimum viable
edge signal.

## Day 1

- Choose one niche pain you personally understand from recent experience.
- Write a one-sentence promise in this format:
  `I turn X into Y in Z minutes without A`.
- Kill the idea immediately if you cannot name one concrete weekly workflow and
  one measurable output.

## Day 2

- Find 3 existing places where this pain is already discussed: Reddit, Discord,
  Slack, GitHub issues, app marketplaces, review sites, or niche forums.
- Save 20 real examples of people complaining about the problem.
- Kill the idea if you cannot collect 20 pain signals in 60 minutes.

## Day 3

- Build a fake-door landing page with one offer, one screenshot or mock, one
  price point, and one CTA.
- Add Stripe payment link or waitlist form.
- Timebox to 90 minutes.
- Kill the idea if the page requires more than one feature explanation to make
  sense.

## Day 4

- Put the offer in one low-friction channel only: a marketplace listing, one
  relevant community post, or replies to existing threads.
- Numeric target: 50 page visits or 5 qualified clicks within 72 hours.

## Day 5 to Day 7

- Build only the concierge or MVP slice needed to fulfill the promise.
- Numeric targets for the week: 1 payment, or 3 explicit replies saying they
  would pay, or 10 waitlist signups from clearly relevant users.
- If none of those happen, stop and pick a new niche rather than polishing the
  product.

# Build Spec

Build only a validation stack, not a full SaaS.

- Frontend: single landing page with headline, screenshot, pricing, and CTA.
- Backend: none unless payment or waitlist capture is needed.
- Payments: Stripe payment link or Gumroad.
- Analytics: simple pageview and CTA tracking.
- Fulfillment: manual or semi-manual workflow behind the scenes.
- AI usage: use AI to compress delivery time for a narrow output, not as the
  product story.
- Time budget: maximum 4 to 6 total hours for the first validation loop. Under
  the user's current 2-hours-per-week rule, this is already tight.

If the user insists on zero outreach, zero content, zero services, and 2 hours
per week, the only acceptable build is a marketplace-listed micro-tool with an
extremely small scope. That is still a low-odds bet for the stated target.

# Kill Criteria

- Kill the 30-day $3000 MRR goal immediately under the current constraint
  bundle.
- Kill any idea where you cannot explain why you have an advantage over a random
  competent developer using ChatGPT.
- Kill any niche where you cannot collect 20 pain signals in 60 minutes.
- Kill any offer that gets fewer than 5 qualified clicks or fewer than 2 intent
  signals in 72 hours after first exposure.
- Kill any MVP that requires more than one core workflow, integration, or
  persona before first payment.
- Kill the no-outreach rule if you want fast revenue. Without some distribution
  action, the plan is not serious.
