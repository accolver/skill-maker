# changelog-generator

> **Status:** Planned

Generate audience-aware changelogs from git history with SemVer classification,
migration notes, breaking change callouts, and dependency update summaries.

## When built, this skill will

- Parse conventional commits to classify changes by SemVer impact
- Write audience-appropriate descriptions (developer vs end-user language)
- Highlight breaking changes with migration instructions
- Group changes by category (features, fixes, performance, dependencies)
- Generate upgrade guides for major version bumps
- Include links to relevant PRs, issues, and documentation

## Predicted delta: +55-65%

Agents produce flat bullet lists of commit messages that don't distinguish
breaking changes from patches, miss migration steps, and use developer jargon
instead of user-facing language.

## Getting started

```
Create a skill for generating audience-aware changelogs with SemVer classification and migration notes
```
